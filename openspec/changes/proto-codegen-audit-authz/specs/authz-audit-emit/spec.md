## MODIFIED Requirements

### Requirement: Authz middleware supports audit recorder injection
`pkg/authz` SHALL provide a `WithAuditRecorder(r *audit.Recorder)` option that injects an audit recorder into the middleware configuration. When the recorder is nil, audit emission SHALL be skipped without error.

#### Scenario: Recorder injected via option
- **WHEN** `Authz(WithFGAClient(c), WithAuthzRulesFunc(rulesFunc), WithAuditRecorder(recorder))` is called
- **THEN** the middleware SHALL hold the recorder for use after each authorization check

#### Scenario: Nil recorder is safe
- **WHEN** `Authz(WithFGAClient(c), WithAuthzRulesFunc(rulesFunc), WithAuditRecorder(nil))` is called
- **THEN** the middleware SHALL function normally without emitting audit events

### Requirement: protoc-gen-servora-authz generates func returning copy

`cmd/protoc-gen-servora-authz` SHALL generate `authz_rules.gen.go` files with:
- An unexported `var _authzRules` holding the rule map
- An exported `func AuthzRules() map[string]authz.AuthzRule` that returns a shallow copy
- The file header `// Code generated by protoc-gen-servora-authz. DO NOT EDIT.`

This is a **BREAKING** change from the current `var AuthzRules` output.

#### Scenario: Generated func returns independent copy

- **WHEN** a caller invokes `AuthzRules()` twice and modifies one copy
- **THEN** the other copy and the internal `_authzRules` SHALL remain unchanged

#### Scenario: Existing IAM rules regenerated as func

- **WHEN** `make api` regenerates `api/gen/go/servora/iam/service/v1/authz_rules.gen.go`
- **THEN** the file SHALL contain `func AuthzRules()` instead of `var AuthzRules`

### Requirement: WithAuthzRules accepts func signature

`pkg/authz` SHALL provide `WithAuthzRulesFunc(fn func() map[string]AuthzRule)` option that accepts the codegen-generated function. The existing `WithAuthzRules(map[string]AuthzRule)` SHALL be retained for backward compatibility with hand-written rules.

#### Scenario: Middleware configured with func rules

- **WHEN** `Authz(WithFGAClient(c), WithAuthzRulesFunc(iampb.AuthzRules))` is called
- **THEN** the middleware SHALL call the function once to obtain the rules map and use it for authorization checks

#### Scenario: Middleware configured with static rules

- **WHEN** `Authz(WithFGAClient(c), WithAuthzRules(staticMap))` is called
- **THEN** the middleware SHALL use the static map directly (backward compatible)

### Requirement: Authz middleware emits authz.decision event after Check
After each authorization Check (whether allowed, denied, or errored), the middleware SHALL call `recorder.RecordAuthzDecision` with the operation name, actor, and an `AuthzDetail` containing Relation, ObjectType, ObjectID, Decision, CacheHit, and ErrorReason.

#### Scenario: Allowed check emits event
- **WHEN** a request passes authorization (Check returns allowed=true)
- **THEN** the middleware SHALL emit an authz.decision event with `Decision: "allowed"` and `CacheHit` reflecting the actual cache status

#### Scenario: Denied check emits event
- **WHEN** a request fails authorization (Check returns allowed=false)
- **THEN** the middleware SHALL emit an authz.decision event with `Decision: "denied"` before returning the permission error

#### Scenario: Check error emits event
- **WHEN** the OpenFGA Check call returns an error
- **THEN** the middleware SHALL emit an authz.decision event with `Decision: "error"` and `ErrorReason` containing the error message

#### Scenario: No recorder skips emission
- **WHEN** the middleware has no recorder configured (nil)
- **AND** a request is processed
- **THEN** authorization SHALL proceed normally without any audit emission

### Requirement: Authz middleware passes full principal to openfga Check
The middleware SHALL construct the full OpenFGA principal string using `string(actor.Type()) + ":" + actor.ID()` before passing it to `openfga.CachedCheck`. It SHALL NOT hardcode any actor type prefix.

#### Scenario: User actor principal construction
- **WHEN** a request from a user actor with Type `"user"` and ID `"abc-123"` is authorized
- **THEN** the middleware SHALL call CachedCheck with user parameter `"user:abc-123"`

#### Scenario: Service actor principal construction
- **WHEN** a request from a service actor with Type `"service"` and ID `"order-svc"` is authorized
- **THEN** the middleware SHALL call CachedCheck with user parameter `"service:order-svc"`

### Requirement: Authz middleware does not hardcode allowed actor types
The middleware SHALL NOT hardcode `a.Type() != actor.TypeUser` as a rejection condition. It SHALL reject `anonymous` actors by default and allow all other actor types (including `service`) to proceed to the OpenFGA Check.

#### Scenario: Service actor allowed to Check
- **WHEN** a service actor makes a request to a CHECK-mode operation
- **THEN** the middleware SHALL construct the principal and proceed with the OpenFGA Check

#### Scenario: Anonymous actor rejected
- **WHEN** an anonymous actor makes a request to a CHECK-mode operation
- **THEN** the middleware SHALL return 403 AUTHZ_DENIED without calling OpenFGA
